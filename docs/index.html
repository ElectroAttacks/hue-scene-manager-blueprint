Home Assistant Blueprint

Hue Scene Refresher

Intelligently refreshes Philips Hue scenes when lights change state, using advanced state transition detection to minimise unnecessary updates.

Overview

Hue Scene Refresher combines three-state evaluation (on / off / partial_on) with transition-aware targeting so only the required rooms and zones are refreshed — never more, never less.

When any monitored Hue entity changes, a short polling window opens, collects every light transition into a single evaluation pass, and provides the full context so targeting stays accurate.

Minimum Home Assistant version: 2025.12.4

Features

⚡

Trigger Monitoring

Watches Hue lights, rooms, or zones for state transitions. Optional supplemental triggers for attribute changes.

🎨

Scene Activation

Configurable transition durations for both scene activation and per-light brightness restoration.

✨

Dynamic Scenes

Per-group dynamic playback toggle with a global speed override for animated Hue effects.

⚙️

Advanced Evaluation

Three-state group assessment, automatic overlapping-zone mapping, and per-entity exclusions.

🔗

Post-Refresh Hooks

Custom actions when a group turns off or after every successful refresh cycle.

🛡️

Safeguards

Burst collection, single-mode execution, and excluded-light force-off prevent ghost states.

Configuration

Trigger Monitoring & Polling

InputDefaultDescription
targets—Hue lights, rooms, or zones to monitor for on/off transitions.
state_trigger_delay1 sPolling interval in seconds — also sets the burst-collection window.
alternative_triggers[]Optional supplemental triggers for attribute changes or custom events.

Scene Activation & Brightness Control

InputDefaultDescription
transition_time2 sDuration used when restoring individual light brightness after scene activation.
scene_transition_time4 sDuration used when activating Hue scenes.
dynamic_scene_speed-1Speed override for dynamic scenes. -1 keeps each scene's default.
targets_with_dynamic_scenes[]Groups that activate scenes with dynamic: true.

Advanced Evaluation & Safeguards

InputDefaultDescription
ignore_active_effectsfalseAllow lights running Hue effects to count toward a group's on state.
turn_off_state_excluded_lightstrueForce-off excluded lights that were off in the snapshot after refresh.
targets_excluded_from_state_check[]Lights excluded from on/off/partial_on assessment.
targets_excluded_from_brightness_restore[]Lights that keep the scene brightness rather than reverting to snapshot level.

Post-Refresh Automation Hooks

InputDefaultDescription
turn_off_actions[]Executed when the evaluated room or zone ends with no active lights.
post_activation_actions[]Executed after scene activation and brightness restoration finish without errors.
Tip: Inside hook actions, use repeat.item[0] for the group entity_id and repeat.item[1] for its snapshot (state, brightness, scene, lights, zones).
